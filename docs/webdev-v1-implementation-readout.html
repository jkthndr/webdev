Dependency map

What should happen first

This is the suggested execution order. Some work can happen in parallel, but these are the integration dependencies to respect.

Start immediately

These reduce friction and can run without waiting on product code.

WEBD-55+ WEBD-56+ WEBD-57

Brief path

Storage unlocks API, API unlocks Studio panel and generation context.

WEBD-58-> WEBD-59-> WEBD-60

Generation path

The create-from-brief shell needs the brief API. The seed can be built in parallel, then plugged in.

WEBD-59-> WEBD-61+ WEBD-68

Preview path

Live preview can start from the existing proxy. Advisory status sits on top of it.

WEBD-62-> WEBD-63

Proof path

The production proof pipeline is the core trust boundary. Failure behavior and timeline UI follow it.

WEBD-64-> WEBD-65-> WEBD-66

Final integration

Feedback context and provenance can land before the final smoke test. The smoke test is the V1 gate.

WEBD-67+ WEBD-69-> WEBD-70
Critical path The tightest path is WEBD-58 -> WEBD-59 -> WEBD-61 -> WEBD-64 -> WEBD-65 -> WEBD-66 -> WEBD-70. That path proves the actual product loop. Everything else should support that, not compete with it.
